Temperature-Based Fan Controller: Revolutionizing Indoor Comfort

1. The Problem We Are Solving

Manually regulating fans to maintain a comfortable indoor environment is inefficient and prone to inconsistencies. Common issues include:

  • Discomfort: Inadequate temperature control leads to an uncomfortable environment.
  • Energy Wastage: Fans often run unnecessarily at high speeds, increasing energy consumption.
  • Health Concerns: Poor temperature and humidity management can contribute to respiratory issues and other health problems.

Traditional methods lack automation, requiring constant attention and adjustment, which is neither practical nor sustainable.

2. Our Proposed Solution: A Temperature-Based Fan Controller

We propose an automated system that uses real-time temperature and humidity data to dynamically adjust fan speed. This solution aims to:

  • Ensure a comfortable indoor atmosphere.
  • Optimize energy usage.
  • Minimize manual intervention.

The system leverages advanced sensors, microcontrollers, and control logic to deliver efficient and intelligent operation.

3. Technology and Working Principle

Core Components and Their Functions:

  1. DHT11 Temperature and Humidity Sensor:
    • Measures ambient temperature and relative humidity.
    • Outputs digital signals representing the environmental conditions.
  2. ESP8266 Microcontroller:
    • A versatile, cost-effective controller with built-in Wi-Fi.
    • Reads and processes data from the DHT11 sensor.
    • Implements control logic for fan speed adjustment using PWM signals.
    • Enables remote monitoring and operation when connected to a Wi-Fi network.
  3. 12V DC Fan:
    • Provides variable airflow.
    • Fan speed is controlled by adjusting the duty cycle of the PWM signal.
  4. Power Supply:
    • Supplies power to the microcontroller, sensor, and fan.

Working Principle:

  1. Sensor Data Acquisition:

    The DHT11 sensor continuously monitors ambient temperature and humidity, providing data to the microcontroller.

  2. Data Processing:

    The ESP8266 interprets sensor data and determines whether adjustments are required.

  3. Control Logic:
    • Temperature Thresholds: Predefined thresholds trigger appropriate fan speed levels.
    • Humidity Thresholds: Optional logic ensures optimal performance in humid environments.
    • PWM Signal Generation: A higher duty cycle increases fan speed, while a lower duty cycle reduces it.
  4. Remote Monitoring and Control (Optional):
    • Users can monitor and adjust the system via a smartphone app or web interface.
    • Notifications and data logs enhance user experience and allow for historical analysis.

4. Advanced Features and Considerations

  1. Adaptive Control Algorithms:

    Adjust fan speed dynamically based on heat load, occupancy, and environmental factors.

  2. Energy Efficiency:
